# Implementation Plan: Race Tab Visualizations

**Branch**: `003-race-tab-visualizations` | **Date**: 2026-04-30 | **Spec**: [spec.md](spec.md)  
**Input**: Feature specification from `specs/003-race-tab-visualizations/spec.md`

## Summary

Replace the Lap Times tab with a Race tab featuring four interconnected charts: a position-changes-per-lap multi-line chart (all drivers, inverted Y-axis), a team pace ranking boxplot (quick laps by team), a single-driver lap time scatterplot coloured by tyre compound (when 1 driver selected), and a multi-driver lap time comparison line chart (when 2+ drivers selected). Driver selection is toggle-based (click a line to add/remove from a Set); clicking a team boxplot highlights that team's drivers across charts. Smooth CSS transitions animate panel and chart state changes. Three new FastAPI endpoints serve the race-specific data; the existing lap-times comparison endpoint is reused for multi-driver mode.

## Technical Context

**Language/Version**: Python 3.11 (backend), JavaScript/React 19 (frontend)  
**Primary Dependencies**: FastAPI + FastF1 (backend); React + MUI v9 + @nivo/line + @nivo/boxplot + @tanstack/react-query + Tailwind CSS v4 (frontend)  
**Storage**: FastF1 disk cache at `.fastf1_cache/` (already configured)  
**Testing**: pytest (backend unit + integration), Vitest + React Testing Library (frontend unit), Playwright (E2E)  
**Target Platform**: Web browser, desktop-first (1280px+), responsive to mobile  
**Project Type**: Web application (React SPA + FastAPI JSON service)  
**Performance Goals**: Race tab reaches usable rendered state within 3s of session data being available  
**Constraints**: Position changes chart handles up to 20 drivers x ~70 laps = ~1,400 data points without frame drops  
**Scale/Scope**: Single-user desktop tool; no multi-tenancy or auth concerns
